## Community Detection Algorithm

| Option | Description | Selected |
|--------|-------------|----------|
| Leiden Algorithm | (Recommended) Standard GraphRAG partitioning; handles hierarchical communities best, but requires complex Rust/C dependencies | ✓ |
| Louvain Algorithm | Simpler graph-based partition, but can produce disconnected or sub-optimal communities | |
| Hierarchical K-Means on node embeddings | Easiest to implement; groups nodes purely by semantic vector similarity, ignoring topology | |
| You decide | Agent will choose the best fit based on implementation complexity | |

**User's choice:** Leiden Algorithm (D-01), Multi-level hierarchy (D-02), In-engine Rust service (D-03), Delta-threshold background async execution (D-04).
**Notes:** The user suggested the "delta threshold approach in the Rust engine: track a counter of dirty nodes (nodes added/removed since last partition), and trigger a background async re-partition task once that counter crosses a configurable threshold." This is captured in D-04.

## Global Query Execution

| Option | Description | Selected |
|--------|-------------|----------|
| Hierarchical selection | (Recommended) Retrieve root community summaries first, identify relevant branches, and traverse down to get detailed leaf summaries if needed | ✓ |
| Flat Map-Reduce | Retrieve summaries for all communities at a specific level of hierarchy, ask LLM to generate intermediate answers in parallel, and reduce to a final answer | |
| Semantic filtering | Vector search the query against embedded community summaries and retrieve only the top-k most relevant community summaries | |
| You decide | Agent will choose the best fit | |

**User's choice:** Hierarchical selection (D-09), Rust RAG engine execution (D-10), Progress event and response token streaming (D-11), Degraded mode fault tolerance (D-12).
**Notes:** Keeps the Go control plane lightweight and unaware of RAG inner map-reduce loops.

| Option | Description | Selected |
