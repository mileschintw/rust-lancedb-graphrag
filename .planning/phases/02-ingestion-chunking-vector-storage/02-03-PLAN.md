# OpenRouter Embeddings & LanceDB Storage (Plan 3)

This plan implements the OpenRouter embedding client with rate limit concurrency throttling and retry backoff, creates the schema layout for LanceDB nodes/edges/communities/documents tables with startup drift detection, and defines the EntityResolver interface.

<threat_model>
### Threat Model Config
ASVS Level: 1. Block on: high severity threats.

- **Threat:** Exposure of Sensitive OpenRouter Credentials.
  - **ASVS Control:** 3.1.1 (Verify that the application does not print credentials in log outputs or debug pages).
  - **Mitigation:** Ensure the OpenRouter API key environment variable is parsed securely and is not logged, printed in error messages, or embedded in stack traces.
- **Threat:** Rate Limiting / API Downtime.
  - **ASVS Control:** 11.1.8 (Verify that application rate limiting is implemented).
  - **Mitigation:** Implement client retry logic (exponential backoff) and restrict maximum concurrency to 5 requests per index job.
</threat_model>

## Tasks

### Task 1: Build OpenRouter Embeddings Client
<read_first>
- `engine/Cargo.toml`
- `engine/src/client/mod.rs`
</read_first>
<acceptance_criteria>
- `engine/Cargo.toml` contains `reqwest = { version = "0.11", features = ["json"] }` and `futures = "0.3"` or similar.
- `engine/src/client/mod.rs` implements `OpenRouterClient` struct and constructor.
- Client implements `async fn get_embeddings(&self, texts: &[String]) -> Result<Vec<Vec<f32>>, String>`:
  - Calls OpenRouter API using model `nvidia/llama-nemotron-embed-vl-1b-v2:free` (producing 2048-dimension embeddings).
  - Batches text inputs and restricts concurrency to 5 parallel HTTP calls using `futures::stream::iter().buffer_unordered(5)`.
  - Retries up to 3 times with exponential backoff starting at 1 second.
- Verification command: `cargo test -p engine client::tests` (verifies retry loops, timeout handling, concurrency boundaries via mocked endpoints).
</acceptance_criteria>
<action>
Add `reqwest` and `futures` to `engine/Cargo.toml`.
Create `engine/src/client/mod.rs` implementing HTTP payload generation, reqwest client invocation, batch execution, and exponential backoff retry loop.
Create `engine/src/client/tests.rs` with mock API servers (using `wiremock` or direct trait mocks) to test client responses under network failure, slow responses, and rate limit errors.
</action>

### Task 2: Configure LanceDB Schemas, Drift Detection, and Entity Resolution
<read_first>
- `engine/Cargo.toml`
- `engine/src/db/mod.rs`
- `engine/src/main.rs`
</read_first>
<acceptance_criteria>
- `engine/Cargo.toml` contains `lancedb = "0.4"` (or compatible embedded) and Arrow dependencies.
- `engine/src/db/mod.rs` implements `DatabaseManager` with connection logic.
- Declares four tables with exact Arrow schemas:
  - `documents`: `document_id` (Utf8), `raw_content` (Binary).
  - `nodes`: `document_id` (Utf8), `chunk_id` (Utf8), `chunk_index` (Int32), `char_start` (Int32), `char_end` (Int32), `content` (Utf8), `embedding` (FixedSizeList(2048, Float32)), `token_estimate` (Int32), `token_estimate_scheme` (Utf8), `token_estimate_version` (Utf8), plus optional nullable columns: `title` (Utf8), `section_path` (Utf8), `page_start` (Int32), `page_end` (Int32), `content_hash` (Utf8), `chunker_version` (Utf8), `embedding_model` (Utf8), `ingested_at` (Int64), `content_type` (Utf8), and placeholder columns: `community_ids` (List(Int32)), `summary` (Utf8), `summary_vector` (FixedSizeList(2048, Float32)), `unsummarized_refs` (List(Utf8)).
  - `edges`: `edge_id` (Utf8), `source_node_id` (Utf8), `target_node_id` (Utf8), `relation_type` (Utf8), `weight` (Float32), `document_id` (Utf8), `summary` (Utf8), `summary_vector` (FixedSizeList(2048, Float32)).
  - `communities`: `community_id` (Int32), `level` (Int32), `title` (Utf8), `summary` (Utf8), `summary_vector` (FixedSizeList(2048, Float32)), `nodes` (List(Utf8)).
- Startup validation: opens LanceDB tables on initialization and verifies schemas. Mismatches cause immediate panic / fast fail.
- Defines async `EntityResolver` trait and implements `ExactMatchResolver` passthrough.
- Verification command: `cargo test -p engine db::tests` (asserts table initialization, schema validation, and drift fast-fail).
</acceptance_criteria>
<action>
Add `lancedb` and Arrow dependencies to `engine/Cargo.toml`.
Create `engine/src/db/mod.rs` defining Arrow schemas for the 4 tables, initialization logic, table checks, and schema field validations.
Define `EntityResolver` trait and `ExactMatchResolver` struct in `engine/src/db/mod.rs`.
Create `engine/src/db/tests.rs` verifying table creation and that mismatched table schemas trigger panic on database initialization.
Register `mod db` and `mod client` in `engine/src/main.rs`.
</action>
